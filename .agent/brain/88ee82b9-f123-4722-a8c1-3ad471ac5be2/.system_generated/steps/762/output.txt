### Create Server Action in Next.js

Source: https://github.com/vercel/next.js/blob/v16.1.6/docs/01-app/02-guides/single-page-applications.mdx

Defines a basic Server Action using the 'use server' directive. Server Actions are async functions that run on the server and can be imported and called from Client Components, eliminating the need to manually create API endpoints.

```typescript
'use server'

export async function create() {}
```

```javascript
'use server'

export async function create() {}
```

### Utilize Closures with Encryption in Next.js Server Actions

Source: https://github.com/vercel/next.js/blob/v16.1.6/docs/01-app/02-guides/data-security.mdx

Server Actions defined within components can form closures, allowing them to access variables from their outer scope. Next.js automatically encrypts these closed-over variables when they are sent to the client and back to the server, providing a secure way to capture a snapshot of data for later use by the action.

```tsx
export default async function Page() {
  const publishVersion = await getLatestVersion();

  async function publish() {
    "use server";
    if (publishVersion !== await getLatestVersion()) {
      throw new Error('The version has changed since pressing publish');
    }
    // ...
  }

  return (
    <form>
      <button formAction={publish}>Publish</button>
    </form>
  );
}
```

### Immediately Expire Cache in Server Actions with updateTag

Source: https://github.com/vercel/next.js/blob/v16.1.6/docs/01-app/01-getting-started/09-caching-and-revalidating.mdx

Uses updateTag to immediately expire cached data within Server Actions for read-your-own-writes scenarios. Unlike revalidateTag, updateTag can only be used in Server Actions and provides immediate cache expiration without stale-while-revalidate support. Multiple tags can be updated in sequence to invalidate related cache entries.

```typescript
import { updateTag } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  // Create post in database
  const post = await db.post.create({
    data: {
      title: formData.get('title'),
      content: formData.get('content'),
    },
  })

  // Immediately expire cache so the new post is visible
  updateTag('posts')
  updateTag(`post-${post.id}`)

  redirect(`/posts/${post.id}`)
}
```

### refresh() - Server Action Function

Source: https://github.com/vercel/next.js/blob/v16.1.6/docs/01-app/03-api-reference/04-functions/refresh.mdx

Refreshes the client router from within a Server Action. This function can only be called from Server Actions and will throw an error if used in Route Handlers, Client Components, or other contexts.

```APIDOC
## refresh()

### Description
Refreshes the client router from within a Server Action. This function revalidates the current page and updates the client-side router state after data mutations.

### Method
Function Call (Server Action)

### Usage Context
Server Actions only - Cannot be used in Route Handlers, Client Components, or other contexts

### Signature
```tsx
refresh(): void;
```

### Parameters
None

### Returns
- **void** - This function does not return a value

### Import
```tsx
import { refresh } from 'next/cache'
```

### Request Example (TypeScript)
```tsx
'use server'

import { refresh } from 'next/cache'

export async function createPost(formData: FormData) {
  const title = formData.get('title')
  const content = formData.get('content')

  // Create the post in your database
  const post = await db.post.create({
    data: { title, content }
  })

  refresh()
}
```

### Request Example (JavaScript)
```js
'use server'

import { refresh } from 'next/cache'

export async function createPost(formData) {
  const title = formData.get('title')
  const content = formData.get('content')

  // Create the post in your database
  const post = await db.post.create({
    data: { title, content }
  })

  refresh()
}
```

### Error Handling

#### Invalid Usage - Route Handler
Calling `refresh()` outside of a Server Action will throw an error:

```tsx
import { refresh } from 'next/cache'

export async function POST() {
  // This will throw an error
  refresh()
}
```

#### Error Conditions
- Called from Route Handlers
- Called from Client Components
- Called from non-Server Action contexts

### Notes
- Must be used within a Server Action (marked with 'use server')
- Typically called after database mutations or data updates
- Provides automatic page revalidation without manual cache management
```
