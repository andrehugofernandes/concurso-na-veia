### Activity Component

Source: https://github.com/reactjs/react.dev/blob/main/src/content/reference/react/Activity.md

The Activity component wraps child UI elements and controls their visibility using the mode prop. When hidden, it applies display: none CSS and destroys Effects. When visible, it restores the previous state and re-creates Effects.

```APIDOC
## Activity Component

### Description
The Activity component hides and restores the UI and internal state of its children. It uses CSS display properties to hide content and manages React Effects lifecycle accordingly.

### Component Signature
```jsx
<Activity mode={visibility}>
  {children}
</Activity>
```

### Props

#### children
- **Type**: ReactNode
- **Required**: Yes
- **Description**: The UI you intend to show and hide.

#### mode
- **Type**: 'visible' | 'hidden'
- **Required**: No
- **Default**: 'visible'
- **Description**: Controls the visibility state of the Activity boundary. When set to 'hidden', React will visually hide children using display: none CSS property and destroy their Effects. When set to 'visible', children are shown with previous state restored and Effects re-created.

### Behavior

#### When Hidden
- Children are visually hidden using `display: none` CSS property
- Effects are destroyed and subscriptions are cleaned up
- Children still re-render in response to new props at lower priority
- No DOM elements are rendered for text-only components

#### When Visible
- Children are revealed with previous state restored
- Effects are re-created
- Full rendering priority is restored

### Usage Example
```jsx
<Activity mode={isShowingSidebar ? "visible" : "hidden"}>
  <Sidebar />
</Activity>
```

### Caveats

- If Activity is rendered inside a ViewTransition and becomes visible as a result of an update caused by startTransition, it will activate the ViewTransition's `enter` animation. If it becomes hidden, it will activate its `exit` animation.
- A hidden Activity that renders only text will not render anything in the DOM, as there is no corresponding DOM element to apply visibility changes to. For example, `<Activity mode="hidden"><ComponentThatJustReturnsText /></Activity>` will produce no output, while `<Activity mode="visible"><ComponentThatJustReturnsText /></Activity>` will render visible text.
```

### App component with Activity boundary for state preservation

Source: https://github.com/reactjs/react.dev/blob/main/src/content/reference/react/Activity.md

Updated example using Activity boundary instead of conditional rendering. The sidebar state is now preserved when toggled hidden/visible.

```jsx
import { Activity, useState } from 'react';

import Sidebar from './Sidebar.js';

export default function App() {
  const [isShowingSidebar, setIsShowingSidebar] = useState(true);

  return (
    <>
      <Activity mode={isShowingSidebar ? 'visible' : 'hidden'}>
        <Sidebar />
      </Activity>

      <main>
        <button onClick={() => setIsShowingSidebar(!isShowingSidebar)}>
          Toggle sidebar
        </button>
        <h1>Main content</h1>
      </main>
    </>
  );
}
```

### Preserve Component State with Activity Component

Source: https://github.com/reactjs/react.dev/blob/main/src/content/blog/2025/04/23/react-labs-view-transitions-activity-and-more.md

Wraps a component in the Activity component with mode toggling to maintain state across page navigation. The 'visible' mode shows the component while 'hidden' keeps it mounted but hidden, preserving all internal state for when the user returns to that page.

```javascript
function App() {
  const { url } = useRouter();

  return (
    <>
      <Activity mode={url === '/' ? 'visible' : 'hidden'}>
        <Home />
      </Activity>
      {url !== '/' && <Details />}
    </>
  );
}
```

### Preserving DOM state with Activity boundaries

Source: https://github.com/reactjs/react.dev/blob/main/src/content/reference/react/Activity.md

By wrapping components in Activity boundaries, the DOM is preserved via CSS display: none instead of being unmounted, maintaining user input across tab switches.

```javascript
import { Activity, useState } from 'react';
import TabButton from './TabButton.js';
import Home from './Home.js';
import Contact from './Contact.js';

export default function App() {
  const [activeTab, setActiveTab] = useState('contact');

  return (
    <>
      <TabButton
        isActive={activeTab === 'home'}
        onClick={() => setActiveTab('home')}
      >
        Home
      </TabButton>
      <TabButton
        isActive={activeTab === 'contact'}
        onClick={() => setActiveTab('contact')}
      >
        Contact
      </TabButton>

      <hr />

      <Activity mode={activeTab === 'home' ? 'visible' : 'hidden'}>
        <Home />
      </Activity>
      <Activity mode={activeTab === 'contact' ? 'visible' : 'hidden'}>
        <Contact />
      </Activity>
    </>
  );
}
```

```javascript
export default function TabButton({ onClick, children, isActive }) {
  if (isActive) {
    return <b>{children}</b>
  }

  return (
    <button onClick={onClick}>
      {children}
    </button>
  );
}
```

### Restoring the state of hidden components

Source: https://github.com/reactjs/react.dev/blob/main/src/content/reference/react/Activity.md

The Activity boundary provides a way to preserve the internal state of a component even when it is visually hidden from the user. By wrapping a component in an Activity boundary and toggling its mode between visible and hidden, the state remains intact without requiring changes to the component's implementation. This approach is particularly useful for UI elements like sidebars where users expect their interactions to persist after toggling visibility.
