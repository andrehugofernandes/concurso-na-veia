### Create Supabase User with Admin API (TypeScript)

Source: https://github.com/supabase/supabase/blob/master/apps/docs/content/guides/platform/migrating-to-supabase/auth0.mdx

This snippet demonstrates how to migrate existing users to Supabase Auth using the `supabase.auth.admin.createUser` method. It provides examples for creating users with either a pre-hashed password (supporting bcrypt and Argon2) or a plaintext password, which Supabase will automatically hash.

```typescript
import { createClient } from '@supabase/supabase-js'
const supabase = createClient('your_project_url', 'your_supabase_api_key')

// ---cut---
const { data, error } = await supabase.auth.admin.createUser({
  email: 'valid.email@supabase.io',
  password_hash: '$2y$10$a9pghn27d7m0ltXvlX8LiOowy7XfFw0hW0G80OjKYQ1jaoejaA7NC',
  email_confirm: true,
})
```

### POST /auth/v1/admin/users - Create User

Source: https://github.com/supabase/supabase/blob/master/apps/www/_blog/2024-11-13-supabase-dynamic-functions.mdx

This endpoint allows administrators to create new user accounts directly through the Supabase Admin API, bypassing standard sign-up flows. It supports setting email, password, and initial user metadata.

```APIDOC
## POST /auth/v1/admin/users

### Description
This endpoint allows administrators to create new user accounts directly through the Supabase Admin API, bypassing standard sign-up flows. It supports setting email, password, and initial user metadata.

### Method
POST

### Endpoint
/auth/v1/admin/users

### Parameters
#### Path Parameters
(None)

#### Query Parameters
(None)

#### Request Body
- **email** (string) - Required - The email address for the new user.
- **password** (string) - Required - The password for the new user.
- **user_metadata** (object) - Optional - Custom metadata to associate with the user.

### Request Example
```json
{
  "email": "user@email.com",
  "password": "password",
  "user_metadata": {
    "name": "Yoda"
  }
}
```

### Response
#### Success Response (200)
- **data** (object) - Contains information about the newly created user, including ID, email, roles, and metadata.
- **error** (object) - Contains error details if the operation failed.

#### Response Example
```json
{
  "data": {
    "id": "a1b2c3d4-e5f6-7890-1234-567890abcdef",
    "email": "user@email.com",
    "aud": "authenticated",
    "role": "authenticated",
    "created_at": "2023-10-27T10:00:00Z",
    "updated_at": "2023-10-27T10:00:00Z",
    "user_metadata": {
      "name": "Yoda"
    }
  },
  "error": null
}
```
```

### Create Supabase Auth User with Metadata (TypeScript)

Source: https://github.com/supabase/supabase/blob/master/apps/docs/content/guides/platform/migrating-to-supabase/auth0.mdx

This TypeScript example demonstrates how to use the `supabase.auth.admin.createUser` method to register a new user. It shows how to assign `user_metadata` for public, user-editable data like `full_name` and `app_metadata` for private, admin-controlled data such as `role`, which are accessible via admin methods. This method requires an initialized Supabase client.

```typescript
import { createClient } from '@supabase/supabase-js'
const supabase = createClient('your_project_url', 'your_supabase_api_key')

const { data, error } = await supabase.auth.admin.createUser({
  email: 'valid.email@supabase.io',
  user_metadata: {
    full_name: 'Foo Bar',
  },
  app_metadata: {
    role: 'admin',
  },
})
```

### Create Supabase Auth User with Custom UUID ID (TypeScript)

Source: https://github.com/supabase/supabase/blob/master/apps/docs/content/guides/platform/migrating-to-supabase/auth0.mdx

This TypeScript example uses an already initialized `supabase` client to call `supabase.auth.admin.createUser` to create a new user with a specified UUID V4 `id`. This is particularly useful when migrating existing users from other systems where their IDs need to be preserved.

```typescript
// specify a custom id
const { data, error } = await supabase.auth.admin.createUser({
  id: 'e7f5ae65-376e-4d05-a18c-10a91295727a',
  email: 'valid.email@supabase.io',
})
```

### Migrate existing users to Supabase Auth

Source: https://github.com/supabase/supabase/blob/master/apps/docs/content/guides/platform/migrating-to-supabase/auth0.mdx

Migrate existing users to Supabase Auth. This requires two main steps: first, check which users need to be migrated, then create their accounts using the Supabase admin endpoints. To prepare, get your Auth 0 user export and password hash export lists. Filter for users who use password login; these users will have `auth0` as a provider under the `identities` field in the user object. You can find their Auth0 `user_id` in the same identity object. Check that the user has a corresponding password hash by comparing their Auth0 `user_id` to the `oid` field in the password hash export. Use Supabase Auth's [admin create user](/docs/reference/javascript/auth-admin-createuser) method to recreate the user. If the user has a confirmed email address or phone number, set `email_confirm` or `phone_confirm` to `true`. Supabase supports bcrypt and Argon2 password hashes. If you have a plaintext password instead of a hash, you can provide that instead; Supabase Auth will handle hashing the password for you, as passwords are **always** stored hashed.
